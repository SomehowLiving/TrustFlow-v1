## Key UI/UX Notes

### Address Book JSON Conversion

When the user fills the **Name** and **Address** columns:

```javascript
// UI State (table rows)
[
  { name: "designer", address: "0xB7BdA0b6a477db6c370B6e8331..." },
  { name: "vendor", address: "0x742d35Cc6634C0532925a3b844..." }
]

// Converted to JSON for signing
{
  "designer": "0xB7BdA0b6a477db6c370B6e8331...",
  "vendor": "0x742d35Cc6634C0532925a3b844..."
}
```

### JSON Preview Section

The **JSON Preview** box on Page 1 shows users:
1. Exactly what format will be signed
2. Transparency into the data structure
3. Helps users understand the conversion

This makes it clear that the address book is **structured data**, not just a form.

### AI Input Benefits

By showing the JSON, you can also:
1. Send it directly to OpenAI for validation
2. Have AI suggest corrections (typos, invalid addresses)
3. Parse complex inputs like CSV uploads

Example AI prompt:
```
User provided this address book JSON:
{
  "designer": "0xB7BdA0b6a477db6c370B6e8331...",
  "vendor": "invalid_address"
}

Validate each address. Return a structured response.
```
